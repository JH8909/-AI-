## 闭环一：选品闭环

选品闭环覆盖从产品录入到确定测试产品的完整流程，包括 6 个功能模块。

---

### F1-1 手动新增产品

| 项目 | 内容 |
|------|------|
| **功能名称** | 手动新增产品 |
| **使用场景** | 运营人员在1688/淘宝看到想测试的产品，手动录入到系统 |
| **用户操作路径** | 产品池 -> 点击"添加产品" -> 填写表单 -> 提交 |
| **输入字段** | 产品名称(必填)、描述、分类(下拉10选1)、价格、成本、标签(逗号分隔)、1688链接(可选)、图片URL(可选) |
| **系统处理逻辑** | 1. 校验必填字段 -> 2. 格式标准化 -> 3. 敏感词检查 -> 4. 写入数据库 |
| **AI调用逻辑** | 无（纯录入，AI评分在后续模块） |
| **输出结果** | 新产品出现在产品池列表，状态=草稿，风险等级=安全 |
| **数据表写入** | products INSERT (status=draft, risk_level=safe, source=manual) |
| **状态流转** | 无（初始为草稿） |
| **前端页面表现** | 弹出Modal表单 -> 提交后Toast提示 -> 列表自动刷新 |
| **异常处理** | 名称重复 -> 提示"产品已存在"；名称含敏感词 -> 自动标记为警告并提示原因 |
| **验收标准** | 1. 点击"添加产品"弹出完整表单 2. 必填项校验（名称不能为空）3. 10个分类可选 4. 提交后产品出现在列表 5. 空价格允许，展示为"-" 6. 敏感词(仿牌/医疗)自动警告 |

---

### F1-2 CSV导入产品

| 项目 | 内容 |
|------|------|
| **功能名称** | CSV批量导入产品 |
| **使用场景** | 运营人员有产品数据表格(xlsx/csv)，批量导入系统 |
| **用户操作路径** | 产品池 -> 点击"CSV导入" -> 选择文件 -> 预览 -> 确认导入 |
| **输入字段** | CSV文件（支持UTF-8/GBK编码），自动识别列名: 产品名称/描述/分类/价格/成本/标签/链接 |
| **系统处理逻辑** | 1. 解析CSV -> 2. 列名映射标准化 -> 3. 逐行校验 -> 4. 去重检查 -> 5. 敏感词过滤 -> 6. 批量写入 |
| **AI调用逻辑** | 无 |
| **输出结果** | 导入成功N条 / 跳过N条(原因列表) / 拦截N条(敏感词) |
| **数据表写入** | products 批量INSERT |
| **状态流转** | 无（初始为草稿） |
| **前端页面表现** | 拖拽区/文件选择 -> 预览表格(前10行) -> 显示统计(总数/通过/拦截) -> 确认按钮 -> 进度条 -> 结果报告 |
| **异常处理** | 编码错误 -> 提示UTF-8/GBK自动检测；空行跳过；价格格式错误跳过；文件过大(>5MB)限制 |
| **验收标准** | 1. 支持CSV文件拖拽上传 2. 自动列映射(中英文列名) 3. 预览前10行数据 4. 显示导入统计(总/成功/失败/拦截) 5. 拦截产品显示关键词原因 6. CSV内仿牌/医疗产品自动拦截 |

### F1-3 产品去重与清洗

| 项目 | 内容 |
|------|------|
| **功能名称** | 产品去重与清洗 |
| **使用场景** | 导入数据后或新增产品后，系统自动执行清洗 |
| **用户操作路径** | 自动触发（新增/导入后后台执行，无需人工操作） |
| **输入字段** | 新导入的产品数据 |
| **系统处理逻辑** | 1. 名称去重(模糊匹配>80%) -> 2. 链接去重(相同source_url) -> 3. 价格格式标准化 -> 4. 分类中文映射(家居->home) -> 5. 标签格式规范 -> 6. 图片URL清洗 |
| **AI调用逻辑** | 无 |
| **输出结果** | 去重后的产品列表 + 重复产品日志 |
| **数据表写入** | products 更新 |
| **状态流转** | 无 |
| **前端页面表现** | 导入结果中显示"已去重N条" |
| **异常处理** | 名称过短(<2字)自动删除；价格>10万自动置空 |
| **验收标准** | 1. CSV导入2个同名不同价产品 -> 只保留1个 2. 中英文分类自动映射 3. 标签标准化(中英文逗号统一) 4. 空名称产品自动过滤 |

### F1-4 AI产品评分

| 项目 | 内容 |
|------|------|
| **功能名称** | AI产品评分 |
| **使用场景** | 对已导入的产品进行多维度AI评分，帮助选品决策 |
| **用户操作路径** | AI评分页面 -> 选择产品(下拉) -> 点击"重新评分" -> 查看5维评分结果 |
| **输入字段** | 产品ID(numeric) |
| **系统处理逻辑** | 1. 从products表读取产品信息 -> 2. 构建评分Prompt -> 3. 调用LLM -> 4. 解析JSON输出 -> 5. 写入product_scores表 |
| **AI调用逻辑** | Prompt: "产品评分(1-10): 市场需求度、利润空间、竞争激烈度、合规风险、内容适配度。JSON输出。" -> 解析LLM返回的JSON |
| **输出结果** | 6个维度评分 + 综合推荐指数 + AI分析理由 + 使用的模型名称 |
| **数据表写入** | product_scores INSERT (product_id, market_demand, profit_margin, competition_intensity, compliance_risk, content_fit, overall_score, reasoning, model_used, tokens_used) |
| **状态流转** | 无（评分记录，不改变产品状态） |
| **前端页面表现** | 评分卡片: 5个进度条 + 综合大数字 + AI分析理由文本框 |
| **异常处理** | LLM超时 -> 提示"AI服务不可用"；JSON解析失败 -> 重试1次；含敏感词 -> 评分前预拦截并提示 |
| **验收标准** | 1. 选择产品后显示评分结果 2. 5个维度各有颜色编码(绿/黄/红) 3. 综合推荐指数醒目显示 4. AI分析理由显示完整 5. 显示使用的模型名称 6. 重新评分按钮可用 7. 无Key时返回Mock评分且明确提示 |

### F1-5 竞品分析

| 项目 | 内容 |
|------|------|
| **功能名称** | 竞品分析 |
| **使用场景** | 对比自选产品和竞品，获取差异化分析和策略建议 |
| **用户操作路径** | 竞品分析页面 -> 选择目标产品 -> 输入竞品链接 -> 点击"开始分析" |
| **输入字段** | 目标产品ID + 竞品链接列表(URL，最多5个) |
| **系统处理逻辑** | 1. 读取目标产品信息 -> 2. 提取竞品信息(标题/价格) -> 3. 构建分析Prompt -> 4. 调用LLM -> 5. 解析JSON -> 6. 写入competitor_analyses表 |
| **AI调用逻辑** | Prompt: "对比{产品}与竞品N个, 分析: 价格对比/差异化/内容策略/综合报告。JSON。" |
| **输出结果** | 价格对比结论 + 差异化分析 + 内容策略建议 + 综合评估报告 + 竞品详情列表 |
| **数据表写入** | competitor_analyses INSERT (product_id, competitor_products JSONB, price_comparison, differentiation, content_strategy, overall_report, model_used) |
| **状态流转** | 无 |
| **前端页面表现** | 左: 价格对比卡片(各竞品价格+优势标签) + 右: 差异化分析 -> 策略建议 -> 综合分析报告 |
| **异常处理** | 竞品链接打不开 -> 跳过该链接继续分析其他；无竞品 -> 生成"产品独特性分析" |
| **验收标准** | 1. 产品下拉选择正常 2. 竞品链接输入框+添加按钮 3. 点击"开始分析"有Loading状态 4. 分析结果展示4个板块 5. 价格对比有卡片展示 6. AI返回对象时自动JSON.stringify 7. 无Key时返回Mock |

### F1-6 高分产品进入测试池

| 项目 | 内容 |
|------|------|
| **功能名称** | 高分产品进入测试池 |
| **使用场景** | AI评分>=7的自动进入"测试池"队列，准备开始内容生产 |
| **用户操作路径** | 自动(也可手动在高分产品点击"加入测试池") |
| **输入字段** | 产品ID |
| **系统处理逻辑** | 1. 检查产品是否有评分 -> 2. 检查overall_score >= 7 -> 3. 更新产品状态为active -> 4. 自动创建一条内容草稿占位 -> 5. 加入内容生成队列 |
| **AI调用逻辑** | 无 |
| **输出结果** | 产品状态变为"测试中"；测试池中新增一条记录 |
| **数据表写入** | products UPDATE (status=active) ; content_drafts INSERT (占位草稿, status=draft) |
| **状态流转** | draft(草稿) -> active(测试中) |
| **前端页面表现** | 产品列表状态更新；AI评分页"加入测试池"按钮(评分>=7时显示) |
| **异常处理** | 已激活产品不可重复加入 |
| **验收标准** | 1. 评分>=7产品显示"加入测试池"按钮 2. 点击后状态变为"测试中" 3. 自动创建内容草稿 4. 低分产品(<7)按钮隐藏或禁用 |

### F2-1 小红书内容生成

| 项目 | 内容 |
|------|------|
| **功能名称** | 小红书内容生成 |
| **使用场景** | 测试池中的产品需要生成小红书种草文案 |
| **用户操作路径** | 内容草稿 -> 点击"生成内容" -> 选择产品 -> 选择平台(小红书) -> AI生成 -> 查看草稿 |
| **输入字段** | 产品ID + 平台(xiaohongshu) + 可选: 风格/语气/字数 |
| **系统处理逻辑** | 1. 读取产品信息(名称/描述/价格/标签) -> 2. 读取已有评分数据 -> 3. 构建Prompt -> 4. 调用LLM -> 5. JSON解析 -> 6. 安全合规检查 -> 7. 写入content_drafts表 |
| **AI调用逻辑** | Prompt: "小红书种草文案。标题15-30字含数字/情绪词, 正文150-300字分段+emoji, 3-6个话题标签。JSON: title, body, hashtags, image_prompt" |
| **输出结果** | 标题 + 正文 + 话题标签 + 图片Prompt |
| **数据表写入** | content_drafts INSERT (product_id, platform=xiaohongshu, title, body, hashtags, image_prompt, status=draft) |
| **状态流转** | draft(草稿) -> pending(待审核) |
| **前端页面表现** | 生成结果卡片: 标题+正文(带emoji)+标签+建议价格+图片Prompt；"提交审核"按钮 |
| **异常处理** | 生成内容含违禁词 -> 自动拦截并标记rejected；LLM返回非JSON -> 重试；生成失败 -> 显示错误原因 |
| **验收标准** | 1. 选择产品+平台后可点击"生成" 2. 生成结果包含4个字段 3. 标题格式规范(含数字/情绪词) 4. 正文含emoji 5. 标签格式正确(带#号) 6. 含违禁词自动拦截 7. 无Key时返回Mock且明确提示 |

### F2-3 图片Prompt生成

| 项目 | 内容 |
|------|------|
| **功能名称** | 图片Prompt生成 |
| **使用场景** | 内容生成时自动附带图片Prompt，供后续设计师/ComfyUI出图 |
| **用户操作路径** | 随内容生成自动附带，也可在内容草稿详情页单独"重新生成图片Prompt" |
| **输入字段** | 产品ID + 平台(影响风格) |

| **AI调用逻辑** | Prompt: "为{产品}生成AI图片描述。小红书: 干净背景+生活场景+自然光。闲鱼: 实拍角度+细节特写。JSON: prompt_cn, prompt_en, style" |

| **数据表写入** | content_drafts UPDATE (image_prompt) |
| **状态流转** | 无 |
| **前端页面表现** | 在内容草稿详情中显示"图片Prompt"区块，含复制按钮 |
| **异常处理** | LLM调用失败 -> 使用模板Prompt |
| **验收标准** | 1. 内容生成时同时生成图片Prompt 2. 中英双语版本 3. 含构图/光线/背景描述 4. 有复制到剪贴板按钮 5. 小红书 vs 闲鱼风格不同 |

### F2-4 合规检查

| 项目 | 内容 |
|------|------|
| **功能名称** | 合规检查（自动执行） |
| **使用场景** | 所有AI生成的内容在提交审核前自动执行合规检查 |
| **用户操作路径** | 自动执行，无需人工触发 |
| **输入字段** | 内容草稿(标题+正文+标签) |
| **系统处理逻辑** | 1. 文本分词 -> 2. 敏感词库匹配(5大类) -> 3. 命中规则判断(warn/block) -> 4. 标记内容状态+风险等级 |
| **AI调用逻辑** | 无（使用规则引擎） |
| **输出结果** | 安全内容正常流转；含warn词标记warning；含block词标记blocked+自动驳回 |
| **数据表写入** | content_drafts UPDATE (risk_level) ; review_queue INSERT with risk_note |
| **状态流转** | pending(待审核) 或 rejected(已拦截) |
| **前端页面表现** | 审核队列中高风险内容显示红色警示标签 |
| **异常处理** | 检测引擎超时 -> 默认安全放行 |
| **验收标准** | 1. 仿牌词(仿牌/原单/复刻) -> block 2. 医疗词(治疗/疗程) -> block 3. 减肥词(瘦身/燃脂) -> block 4. 保健品词 -> block 5. 边缘词(Nike/Adidas) -> warn 6. 安全内容正常放行 |

---

### F2-5 人工审核队列

| 项目 | 内容 |
|------|------|
| **功能名称** | 人工审核队列（核心安全闸门） |
| **使用场景** | 运营人员每日审阅AI生成的内容，决定哪些可以发布 |
| **用户操作路径** | 审核队列 -> 浏览待审内容 -> 点击"审核" -> 查看全文 -> 选择通过/驳回(附理由) |
| **输入字段** | 审核操作: 内容ID + 审核状态(approved/rejected) + 审核意见(驳回时必填) |
| **系统处理逻辑** | 1. 读取待审核内容 -> 2. 展示完整内容 -> 3. 接收审核操作 -> 4. 更新content_drafts状态 -> 5. 更新review_queue状态 -> 6. 记录审核日志 |
| **AI调用逻辑** | 无（纯人工决策） |
| **输出结果** | 通过 -> 内容状态变为approved；驳回 -> 状态变为rejected + 审核意见 |
| **数据表写入** | content_drafts UPDATE (status, review_comment) ; review_queue UPDATE (status, reviewer_id, comment, reviewed_at) ; audit_logs INSERT |
| **状态流转** | pending -> approved / rejected |
| **前端页面表现** | 卡片列表: 产品名+平台+标题+内容摘要+状态标签；点击审核展开: 全文+审核意见输入框+通过/驳回按钮 |
| **异常处理** | 驳回时未填写理由 -> 按钮禁用并提示 |
| **验收标准** | 1. 待审列表展示完整 2. 点击审核展开全文 3. 通过/驳回按钮可用 4. 驳回时意见为空按钮禁用 5. 通过后内容从队列移除 6. 高风险内容红色标记 7. 审核记录可追溯 |

### F2-6 审核通过 -> 发布排期

| **输出结果** | 内容状态变为"已排期" + 显示计划发布时间 |
| **数据表写入** | content_drafts UPDATE (status=scheduled, scheduled_at) |

### F3-1 发布数据录入

| 项目 | 内容 |
