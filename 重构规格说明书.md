# 系统重构规格说明书 v0.3.0

> 日期: 2026-06-24
> 涉及: 10项核心重构

---

## 1. 架构原则变更

### 1.1 数据优先级

```
改前: Mock(默认) -> Supabase(可选)
改后: Supabase(优先) -> Mock(仅Supabase不可用时)
```

### 1.2 实现方式

所有 API 路由改为:
1. 先尝试 Supabase 连接
2. Supabase 可用 -> 使用 Supabase 数据
3. Supabase 不可用 -> 降级到 Mock (带提醒)

---

## 2. 产品状态流转变更

### 2.1 旧状态 (6种)
draft -> active -> released -> scaling / paused / archived

### 2.2 新状态 (9种)
draft(草稿) -> testing_candidate(测试候选)
  -> content_ready(内容就绪) -> review_pending(审核中)
  -> published(已发布) -> tracking(追踪中)
  -> scale(放大) / optimize(优化) / rejected(淘汰)

### 2.3 状态迁移图
```
draft ──▶ testing_candidate ──▶ content_ready ──▶ review_pending
                                                       │
                                              ┌────────┴────────┐
                                              │                 │
                                         published          rejected
                                              │
                                         tracking ──▶ scale
                                                     ▶ optimize
```

## 3. 数据库表变更

### 3.1 publishing_records 表

```sql
CREATE TABLE publishing_records (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  product_id UUID REFERENCES products(id),
  platform TEXT NOT NULL,       -- xiaohongshu / xianyu / douyin
  publish_url TEXT,
  publish_time TIMESTAMPTZ,
  title TEXT,
  content TEXT,
  price NUMERIC(10,2),
  status TEXT DEFAULT 'draft',  -- draft / published
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 3.2 data_snapshots 扩展字段

新增字段: favorites, comments, dms, consultations, deals, revenue, profit, refunds

当前前端日数据录入表单仍只开放 views、likes、shares、sales_estimate 4 个基础指标；以上字段已在数据库层预留。

### 3.3 product_scores 扩展

新增: supply_chain_stability (1-10), visual_packaging (1-10)

## 5. 审核清单制

### 5.1 高风险内容判定

以下情况自动标记为高风险，禁止快速通过:
- 产品风险等级为 warning 或 blocked
- 内容含边缘敏感词(品牌名等)
- 建议价格 > 成本 5 倍
- 首次上架的产品内容

### 5.2 审核清单

每次审核必须勾选的检查项:
- [ ] 产品合规: 非仿牌/医疗/减肥/保健品
- [ ] 内容真实: 无虚假宣传
- [ ] 价格合理: 在市场范围内
- [ ] 文案质量: 语句通顺
- [ ] 标签准确: 话题标签相关
