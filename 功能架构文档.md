## 一、项目概览

### 1.1 定位

AI 电商选品中台是面向电商运营人员的内部工具，实现从产品数据采集、清洗、AI 评分、竞品分析，到内容生成、人工审核、数据复盘的完整链路。同时集成 1688 爆品雷达模块，支持关键词监控、任务队列、每日 Top 20 和通知推送。

### 1.2 技术栈

| 层级 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Next.js | 14.2.35 |
| UI 库 | React + Shadcn/ui | 18.3.1 |
| 样式 | Tailwind CSS | 3.x |
| 数据库 | Supabase (PostgreSQL) | - |
| AI Provider | DeepSeek / GPT / Claude | 统一适配 |
| 采集引擎 | Python + Playwright | 3.12 |
| 自动化编排 | n8n | - |
| 包管理 | npm (workspaces) | 11.x |
| 构建工具 | Turborepo | 2.x |

### 1.3 数据流

```

│  仪表盘 → 产品池 → 爆品雷达 → AI评分 → 竞品分析 → 内容生成 → 审核 → 复盘  │
```

## 三、API 端点

### 3.1 业务数据接口

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| GET | `/api/products` | `search`, `category`, `status` | 产品列表 (支持筛选) |
| GET | `/api/scores` | `product_id` | 评分列表 |
| GET | `/api/content-drafts` | `platform` | 内容草稿列表 |
| GET | `/api/review-queue` | - | 待审核队列 |
| PATCH | `/api/review-queue` | `id`, `status`, `comment` | 审核操作 |
| GET | `/api/data-recap` | `product_id` | 7天复盘数据 |
| GET/POST | `/api/settings` | 配置JSON | 系统配置读写 |

### 3.2 AI 接口

| 方法 | 路径 | 说明 | 需要 |
|------|------|------|------|

| POST | `/api/ai/generate` | 内容生成(小红书/闲鱼) | DeepSeek Key |

所有 AI 接口自动降级: 有 Key → 真实 AI, 无 Key → Mock 数据

| GET/POST | `/api/hot-radar/runs` | 抓取任务管理 |
| POST | `/api/hot-radar/score` | 产品评分生成 |
| POST | `/api/hot-radar/expand` | AI 关键词扩展 |
| PATCH | `/api/hot-radar/products/[id]` | 候选产品操作(选入/忽略) |
| POST | `/api/hot-radar/notifications/send` | 榜单推送通知 |

## 四、数据模型

### 4.1 核心表 (schema.sql)

```
products         产品主表              (id, name, category, price, cost, images, specs, risk_level)
product_scores   AI评分记录            (product_id → products)
competitor_analyses 竞品分析           (product_id → products)
content_drafts   内容草稿              (product_id → products, platform)
review_queue     审核队列              (content_draft_id → content_drafts)
data_snapshots   数据快照              (product_id, snapshot_date)
audit_logs       操作审计日志           (table_name, record_id, action)
risk_keywords    敏感词库              (keyword, category, action)
```

### 4.2 爆品雷达表 (migration)

```
monitor_keywords      监控关键词           (keyword, kind, enabled)
crawl_jobs            抓取任务队列          (keyword_id → monitor_keywords, status, cycle_at)
product_observations  产品观测数据          (crawl_job_id → crawl_jobs, name, price, sales)
hot_product_scores    每日爆品评分           (observation_id → product_observations, score_date)
notification_deliveries 通知推送记录        (channel, status)
```

## 六、安全体系

| 安全层级 | 措施 | 说明 |
|---------|------|------|
| 内容安全 | 敏感词过滤库 | `safety.py` 5大类关键词拦截 |
| AI输出 | JSON格式强制 + 安全校验 | generator.ts 自动检测违禁内容 |
| 审核流程 | 人工审核队列 | 所有AI内容必须通过审核 |
| 自动发布 | 禁用 | 代码层无自动发布到任何平台的功能 |
| 数据库 | RLS (Row Level Security) | Supabase 所有表已启用RLS |

## 七、自动化编排

### 7.1 n8n 工作流 (9个)

| 工作流 | 触发方式 | 说明 |
|--------|----------|------|
| product-import | Webhook POST | 产品导入 → 清洗 → Supabase写入 |
| ai-scoring | Webhook POST | 接收产品ID → AI评分 → 保存评分 |
| competitor-analysis | Webhook POST | 接收产品ID+竞品 → AI分析 → 保存 |
| content-generation | Webhook POST | 接收产品ID → AI生成 → 保存草稿 → 加入审核 |
| review-pipeline | 定时(每小时) | 检查待审核 → 飞书通知 |
| daily-snapshot | 定时(每日0点) | 活跃产品数据快照 |
| weekly-recap | 定时(每周一) | 7天数据 → AI复盘报告 |
| hot-radar-daily-ranking | 定时(每日) | 爆品每日榜单生成 |
| hot-radar-six-hour | 定时(每6小时) | 关键词抓取任务创建 |
