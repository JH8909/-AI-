> 版本: v0.2.0 | 2026-06-24
> 系统地址: http://localhost:3001
> 维护团队: AI 电商中台开发组

## 第一章：系统概述

### 1.1 系统定位

AI 电商选品中台是一款面向电商运营人员的选品决策辅助系统，覆盖从产品采集到复盘的全链路。系统采用 AI 驱动 + 人工审核的安全设计理念，所有 AI 生成内容均需经过人工审核后才能发布。

### 1.2 核心能力

| 能力 | 说明 |
|------|------|
| 选品决策 | AI多维度评分 + 竞品分析，辅助判断产品潜力 |
| 内容生成 | AI自动撰写小红书种草文案、闲鱼上架文案 |
| 安全审核 | 敏感词自动拦截 + 人工审核队列，双重保障 |
| 数据复盘 | 7天趋势追踪 + AI复盘报告 + 智能决策建议 |
| 爆品雷达 | 1688关键词监控 + 每日TOP20榜单 + 通知推送 |

### 1.3 适用角色

| 角色 | 使用场景 |
|------|----------|
| 电商运营 | 选品决策、内容生产、数据复盘 |
| 内容创作者 | 获取文案灵感、管理发布排期 |
| 团队管理者 | 审核内容质量、查看整体数据 |

## 第二章：快速开始

### 2.1 首次访问

打开浏览器访问 http://localhost:3001，进入系统仪表盘。

### 2.2 配置 AI 能力（推荐）

系统默认使用 Mock 数据，配置 DeepSeek API Key 后可启用真实 AI 能力。

操作路径: 设置 -> LLM 模型配置
1. 在 DeepSeek API Key 输入框中填入你的 Key
2. 选择默认模型（推荐 deepseek-chat）
3. 点击"保存配置"
4. 点击"测试连接"验证配置是否成功

> 获取 API Key: https://platform.deepseek.com/api_keys

### 2.3 配置数据库（可选）

如需数据持久化，需配置 Supabase 数据库。

操作路径: 设置 -> 数据库配置
1. 填入 Supabase URL（格式: https://xxx.supabase.co）
2. 填入 Supabase Anon Key
3. 点击"保存数据库配置"

> 数据库初始化需在 Supabase Dashboard 中执行迁移SQL:
> packages/supabase-schema/schema.sql (核心8表)
> packages/supabase-schema/migrations/20260624_hot_product_radar.sql (雷达5表)

## 第三章：业务操作流程

### 3.1 标准工作流

```
添加产品 -> AI评分 -> 内容生成 -> 审核通过 -> 发布记录 -> 每日数据 -> 复盘决策
```

### 3.2 第一步：产品池管理

**入口:** 左侧导航栏 -> 产品池

**添加单个产品:**
1. 点击"添加产品"按钮
2. 填写产品名称(必填)、描述、分类、价格、成本、标签、1688链接
3. 点击"保存"

**批量导入:**
1. 点击"CSV导入"按钮
2. 拖拽或点击选择 CSV 文件
3. 预览前10行数据，确认列映射正确
4. 查看导入统计（成功/跳过/拦截）
5. 点击"导入 N 条产品"确认导入

**CSV格式要求:**
```
产品名称,描述,分类,价格,成本,标签,链接
小夜灯USB充电,温馨柔光护眼,家居,25.00,8.00,小夜灯,卧室,https://detail.1688.com/xxx
```

> 注意: 含仿牌/医疗/减肥/保健品关键词的产品会被自动拦截

**筛选与搜索:**
- 搜索框: 输入产品名称关键词实时筛选
- 分类下拉: 10个分类选项
- 状态下拉: 草稿/已上架/已归档

### 3.3 第二步：AI 评分

**入口:** 左侧导航栏 -> AI 评分

**操作:**
1. 从产品下拉列表选择需要评分的产品
2. 系统显示该产品的历史评分（如有）
3. 点击"重新评分"按钮触发 AI 评分

**评分维度说明:**

| 维度 | 范围 | 越高越好? | 说明 |
|------|------|-----------|------|
| 市场需求度 | 1-10 | 是 | 搜索热度/消费趋势 |
| 利润空间 | 1-10 | 是 | 售价-成本的利润空间 |
| 竞争激烈度 | 1-10 | 否 | 市场竞争程度 |
| 合规风险 | 1-10 | 否 | 仿牌/医疗/违规风险 |
| 内容适配度 | 1-10 | 是 | 适合做内容营销的程度 |

**高分产品操作:**
- 评分 >= 7 的产品会显示"加入测试池"按钮
- 点击后产品状态变为"测试中"，并自动创建内容草稿占位

### 3.4 第三步：竞品分析

**入口:** 左侧导航栏 -> 竞品分析

**操作:**
1. 选择目标产品
2. 在竞品链接输入框中输入竞品商品链接，点击"+"添加
3. 点击"开始分析"按钮
4. 系统自动解析竞品信息并调用 AI 生成分析报告

**分析报告包含:**
1. 价格对比卡片: 各竞品价格 + 优势标签
2. 差异化分析: 核心卖点差异
3. 内容策略建议: 小红书/闲鱼内容方向
4. 综合分析报告: 综合评估和行动建议

### 3.5 第四步：内容生成

**入口:** 左侧导航栏 -> 内容草稿

**生成内容:**
1. 点击"生成内容"按钮
2. 在弹出的选择器中:
   - 选择产品（下拉选择已有产品）
   - 选择平台（小红书/闲鱼）
3. 点击"生成"
4. 系统调用 AI 生成文案

**生成结果展示:**
- 标题（小红书含数字/情绪词，闲鱼精简）
- 正文（小红书含 emoji，闲鱼卖点突出）
- 话题标签（小红书 3-6 个）
- 图片 Prompt（中英文双语，可复制）
- 建议价格（限闲鱼）

**提交审核:**
- 生成完成后点击"提交审核"
- 内容自动经过合规检查
- 安全内容进入审核队列
- 含违禁词内容自动驳回

**平台筛选:**
- 点击"全部": 显示所有平台
- 点击"小红书": 仅显示小红书文案
- 点击"闲鱼": 仅显示闲鱼文案

### 3.6 第五步：审核队列

**入口:** 左侧导航栏 -> 审核队列

> 安全原则: 所有 AI 生成内容必须经过人工审核

**快速审核:**
1. 浏览待审内容列表
2. 点击"快速通过": 直接通过（适合低风险内容）
3. 文章从队列移除，状态变为"已通过"

**详细审核:**
1. 点击"审核"按钮展开审核面板
2. 查看完整内容
3. 填写审核意见（驳回时必填）
4. 点击"通过"或"驳回"

**风险提示:**
- 高风险内容（美容器具等）自动显示红色警示标签
- 含敏感词(仿牌/医疗)自动标记并拦截

### 3.7 第六步：发布排期

**入口:** 内容草稿 -> 已通过列表

**操作:**
1. 审核通过的内容会显示"已通过"标签
2. 点击"排期发布"按钮
3. 在弹出的输入框中输入排期时间（格式: YYYY-MM-DD HH:mm）
4. 确认后内容状态变为"已排期"

### 3.8 第七步：数据录入

**入口:** 产品池 -> 产品行操作

**记录发布数据:**
1. 在产品列表中找到已发布的产品
2. 点击该行的"发布"按钮
3. 选择发布日期、发布平台、发布链接
4. 点击保存

**录入每日数据:**
1. 在产品列表中找到已发布的产品
2. 点击该行的"日数据"按钮
3. 选择日期，填写浏览量、点赞量、分享量、预估销量
4. 点击保存

### 3.9 第八步：7天复盘

**入口:** 左侧导航栏 -> 7天复盘

**查看数据:**
1. 选择复盘产品
2. 查看 4 个核心指标的 7 天趋势:
   - 浏览量（7天涨幅%）
   - 点赞量（7天涨幅%）
   - 分享量（7天涨幅%）
   - 预估销量（7天涨幅%）
3. 查看每日数据明细表

**生成 AI 复盘报告:**
1. 点击"生成AI复盘报告"按钮
2. AI 分析 7 天数据，生成 5 段报告:
   - 总结: 一句话概括表现
   - 曝光分析: 浏览量趋势解读
   - 互动评估: 点赞/分享率分析
   - 转化分析: 销量转化评估
   - 风险提示: 异常波动预警
   - 优化建议: 可执行的改进方向

**生成决策建议:**
1. 点击"生成决策建议"按钮
2. AI 根据数据自动判断决策类型:
   - 放大投入(绿色): 曝光增长>50% + 互动率>10%
   - 优化调整(黄色): 数据平稳或轻微下降
   - 淘汰下架(红色): 曝光下降>30% + 互动率<3%
3. 查看详细的行动方案

**应用决策:**
1. 点击"应用决策"按钮
2. 产品状态自动更新(scaling/paused/archived)

**生成执行任务:**
1. 点击"生成任务"按钮
2. 系统根据决策类型生成 4 条预设任务
3. 可在列表中勾选完成/编辑任务

## 第四章：爆品雷达（可选模块）

### 4.1 功能概述

1688 爆品雷达模块用于监控关键词热度，自动生成每日 TOP20 爆品榜单，支持通知推送。

### 4.2 操作流程

**Step 1: 管理关键词**
1. 在爆品雷达页面的关键词输入框中输入种子词（逗号分隔，最多10个）
2. 点击"保存关键词"
3. 可选: 点击"AI 扩展"让 AI 生成关联关键词

**Step 2: 创建抓取任务**
- 点击"创建抓取任务"
- 系统为每个关键词创建抓取任务
- 任务列表显示每个任务的状态(pending/running/completed)

**Step 3: 生成 TOP20**
- 点击"生成 Top 20"
- 系统调用 AI 对候选产品进行评分
- 榜单按综合评分排序展示

**Step 4: 查看榜单**
- 榜单表格显示: 排名、产品名称、分数、信心值、来源词
- 每个产品可操作"选入"(进入产品池)或"忽略"

**Step 5: 推送通知**
- 点击"推送榜单"发送到配置的 Webhook
- 支持企业微信和飞书

## 第六章：常见问题

### Q1: 页面显示空白或加载失败？

A: 刷新页面即可。如果持续失败，重启服务器:
```bash
# 杀掉 Node 进程后重新启动
taskkill /F /IM node.exe
```

### Q2: AI 返回 Mock 数据？

A: 说明 DeepSeek API Key 未配置或配置有误。请检查:
1. 设置页面是否已保存 API Key
2. Key 是否有效（点击"测试连接"验证）
3. 模型选择是否正确

### Q3: 爆品雷达按钮全部禁用？

A: 页面正在加载数据。等待 <1 秒后自动恢复。

### Q4: 如何导入真实产品数据？

A: 使用 Python 采集脚本:
```bash
cd services/scraper
pip install -r requirements.txt
playwright install chromium
python main.py import --file products.csv
python main.py parse --url "https://detail.1688.com/xxx"
```

### Q5: 如何执行数据库迁移？

A: 在 Supabase Dashboard 的 SQL Editor 中执行:
1. packages/supabase-schema/schema.sql (核心8表)
2. packages/supabase-schema/migrations/20260624_hot_product_radar.sql (雷达5表)

### Q6: 端口3000被占用？

A: 系统默认使用 3001 端口。如需修改:
```bash
node node_modules/next/dist/bin/next dev -p <端口号>
```

## 附录：状态流转参考

```
产品状态:
  draft(草稿) -> active(测试中) -> released(已发布) -> scaling(放大)
                                                        -> paused(优化)
                                                        -> archived(淘汰)

内容草稿状态:
  draft(草稿) -> pending(待审核) -> approved(已通过) -> scheduled(已排期)
                                  -> rejected(已驳回)

审核队列状态:
  pending(待审核) -> approved(已通过)
                   -> rejected(已驳回)
```
