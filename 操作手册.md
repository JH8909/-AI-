访问地址: http://localhost:3001

## 一、基础设置

### 1.1 配置 DeepSeek API Key（首次必做）

设置 → 填入 DeepSeek API Key → 选择模型 → 保存配置 → 测试连接

| 字段 | 说明 | 获取方式 |
|------|------|----------|
| DeepSeek API Key | 必填。真实AI评分/内容生成需要 | platform.deepseek.com/api_keys |
| 默认模型 | 推荐 deepseek-chat 或 deepseek-reasoner(R1) | - |
| API Base URL | 留空即用 DeepSeek 官方 | 可替换为其他兼容 OpenAI 的 API |

保存后立即生效，无需重启服务。

### 1.2 测试连接

点击"测试连接"按钮，系统将调用 AI 评分 API 验证配置：
- Mock模式（未配置Key）：未检测到API Key
- Key无效：DeepSeek API 错误: Authentication Fails...
- Key有效：AI响应成功！综合评分: X.X/10

## 二、完整操作流程

产品池 → AI评分 → 竞品分析 → 内容生成 → 审核 → 复盘

### 步骤 1：产品池

入口：左侧导航栏 → 产品池

功能：
- 搜索：输入产品名称实时筛选
- 分类筛选：服饰/电子/家居/美妆/食品/运动/玩具/图书/数码/其他
- 状态筛选：草稿/已上架/已归档
- 风险标签：安全(绿) / 警告(黄) / 已拦截(红)

操作：
搜索框输入关键词 → 选择分类 → 选择状态 → 查看结果表格

表格字段：产品名称 | 品类 | 价格 | 成本 | 来源 | 风险 | 状态 | 创建日期

---

### 步骤 2：AI 评分

入口：AI 评分

功能：对产品进行5维AI评分

| 维度 | 范围 | 说明 |
|------|------|------|
| 市场需求度 | 1-10 | 越高越好 |
| 利润空间 | 1-10 | 越高越好 |
| 竞争激烈度 | 1-10 | 越低越好 |
| 合规风险 | 1-10 | 越低越好 |
| 内容适配度 | 1-10 | 越高越好 |
| 综合推荐指数 | 0-10 | 加权汇总 |

操作：
选择产品 → 查看评分卡片(含进度条) → 阅读AI分析理由 → 点击重新评分

安全红线（AI自动识别）：
- 仿牌/山寨 → 自动低分
- 医疗/减肥/保健品 → 自动低分

---

### 步骤 3：竞品分析

入口：竞品分析

功能：对比目标产品与竞品，生成分析报告

操作：
选择目标产品 → 输入竞品链接 → 点击开始分析

输出报告：
1. 价格对比 - 目标产品与竞品价格对比+分析结论
2. 差异化分析 - 核心卖点差异+市场定位分析
3. 内容策略建议 - 小红书/闲鱼内容方向建议
4. 综合分析报告 - 综合评估结论和行动建议

---

### 步骤 4：内容生成

入口：内容草稿

功能：AI生成小红书种草文案或闲鱼上架文案

操作：
点击生成内容 → 输入产品名称 → 系统调用AI → 自动生成草稿

平台筛选：
点击全部 → 显示所有
点击小红书 → 仅显示小红书文案
点击闲鱼 → 仅显示闲鱼文案

草稿状态：
- 待审核(黄) - 等待人工审核
- 已通过(绿) - 审核通过可用
- 已驳回(红) - 需重新生成

操作按钮：
- 查看详情 - 展开查看完整内容
- 提交审核 - 将草稿提交到审核队列
- 已通过 - 已审核通过（只读）

---

### 步骤 5：审核队列

入口：审核队列

安全原则：所有内容必须人工审核后才能使用。仿牌/医疗/减肥/保健品直接驳回。

快速通过：
点击快速通过 → 内容状态变为已通过

详细审核：
点击审核 → 展开审核面板
- 输入审核意见（驳回时必填）
- 点击通过 → 审核通过
- 点击驳回 → 输入理由后驳回（理由为空时按钮禁用）
- 点击取消 → 关闭审核面板

风险提示：高风险产品自动添加红色风险提示标签。

---

### 步骤 6：7天数据复盘

入口：数据复盘

操作：
选择复盘产品 → 查看4个核心指标趋势
- 浏览量(7天涨幅%)
- 点赞量(7天涨幅%)
- 分享量(7天涨幅%)
- 预估销量(7天涨幅%)
→ 查看每日数据明细表
→ 阅读AI复盘报告
→ 点击导出报告

AI复盘报告包含：
1. 曝光趋势分析
2. 互动表现评估
3. 转化数据分析
4. 风险预警
5. 运营优化建议

## 四、API端点一览

|------|------|------|

## 六、常见问题

Q: 页面显示Build Error？
A: 刷新页面即可。首次启动需等待.next缓存重建。

Q: 端口3000被占用？
A: 使用 -p 3001 或其他端口启动。

Q: 如何导入真实产品数据？
A: cd services/scraper && python main.py import --file products.csv
