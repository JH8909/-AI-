## 三、业务闭环架构

### 闭环一：选品闭环 (7个模块)

```ascii
手动新增 ──▶ CSV导入 ──▶ 去重清洗 ──▶ AI评分(7维) ──▶ 竞品分析 ──▶ 进入测试池
    │              │             │              │              │              │
    └── 一键提取 ──┘             自动执行       供应链+视觉    结构化报告       status变更
    (1688链接)
```

| 模块 | 页面 | 输入 | 输出 |
|------|------|------|------|
| 一键提取链接 | 添加产品弹窗 | 1688商品URL | 名称/描述/分类/价格/标签自动填充 |
| 手动新增 | 产品池弹窗 | 表单填写 | products表 INSERT |
| CSV导入 | 产品池弹窗 | CSV文件(UTF-8/GBK) | products表 批量INSERT |
| 去重清洗 | API自动 | 新导入数据 | 模糊匹配去重(>80%)+分类映射 |
| AI 7维评分 | AI评分页 | 产品信息 | 市场/利润/竞争/合规/内容/供应链/视觉 |
| 竞品分析 | 竞品分析页 | 产品+竞品URL | 推荐售价/关键词/差评风险/内容角度 |
| 进入测试池 | AI评分页按钮 | 产品ID(score>=7) | status->testing_candidate + 创建草稿 |

### 闭环二：内容闭环 (6个模块)

```ascii
内容生成选择器 ──▶ 内容资产包 ──▶ 合规检查 ──▶ 人工审核 ──▶ 发布排期
      │               │               │             │             │
  选择产品+平台   小红书+闲鱼       敏感词拦截     清单制5项     日期时间
                  +Prompt+回复       (blocked)     (高风险禁快速)
```

| 模块 | 输入 | 输出 |
|------|------|------|
| 内容资产包 | 产品+平台 | xiaohongshu + xianyu + image_prompt + reply_templates |
| 合规检查 | 标题+正文+标签 | blocked(自动拦截) / warning / safe |
| 审核清单 | 5项检查项勾选 | 高风险内容必须全选才能通过 |
| 发布排期 | 日期时间 | content_drafts.status=scheduled |

### 闭环三：复盘闭环 (6个模块)

```ascii
发布录入 ──▶ 日数据(当前表单4字段/数据库13字段) ──▶ AI复盘报告 ──▶ 决策建议 ──▶ 状态更新 ──▶ 任务生成
   │              │                 │              │             │             │
日期+平台+链接  浏览/点赞/分享     5段分析       magnify/optimize 当前映射     4条预设
                预估销量                         /eliminate     scale/optimize/rejected
                数据库预留更多指标
```

### 闭环四：爆品雷达 (独立模块)

关键词管理 → 创建任务 → 生成TOP20 → 榜单展示 → 通知推送(企微/飞书)

## 七、状态流转

```ascii
产品状态(9种):
draft → testing_candidate → content_ready → review_pending
                                               ↓
                                         published → tracking → scale
                                                              → optimize
                                         rejected (审核不通过或拦截)

内容草稿状态:
draft → pending → approved → scheduled
                 → rejected

审核队列状态:
pending → approved (清单通过)
         → rejected (附理由)
```

## 八、安全体系

| 层级 | 措施 | 说明 |
|------|------|------|
| 内容安全 | 敏感词过滤(6大类) | 仿牌/医疗/减肥/保健品/三无/品牌 |
| 合规检查 | 内容创建时自动执行 | blocked/warning/safe |
| 审核清单 | 高风险内容禁止快速通过 | 必须勾选5项检查项 |
| 自动发布 | 禁用 | 无自动发布功能 |
| 数据持久化 | review_queue双写 | Supabase优先+本地缓存兜底+同步机制 |
